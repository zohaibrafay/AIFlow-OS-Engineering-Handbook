# Master Context

## Project Summary

AIFlow OS Engineering Handbook is a documentation-first repository for designing an AI-native workflow automation platform. It will eventually guide implementation of workflow authoring, execution, scheduling, integrations, plugin development, AI runtime services, MCP capabilities, enterprise governance, cloud operations, and marketplace distribution.

## Repository Structure

The repository is organized into root governance files, `docs/` for project-wide documentation, `volumes/` for handbook content, `adr/` for decision records, `templates/` for reusable structures, `prompts/` for AI-assisted workflows, `diagrams/` for architecture visuals, `assets/` for supporting media, `examples/` for future reference material, and `.github/` for collaboration workflows.

## Architecture Summary

The repository separates documentation governance from future platform implementation. The handbook will define platform architecture before code is introduced. Future implementation repositories or directories should align with approved roadmap items, ADRs, standards, and glossary terms.

## Technology Decisions

- Markdown is the primary documentation format.
- Mermaid and C4-style diagrams are the expected diagram foundations until a different decision is approved.
- Semantic versioning governs handbook releases.
- ADRs will govern material architecture decisions.
