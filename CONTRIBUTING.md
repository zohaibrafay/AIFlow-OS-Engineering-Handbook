# Contributing

## Contribution Model

AIFlow OS Engineering Handbook is documentation-first. Contributions should improve the repository as an engineering reference, not as marketing collateral or speculative product copy.

## Before You Start

Read these files before opening an issue or pull request:

- [docs/MASTER_CONTEXT.md](docs/MASTER_CONTEXT.md)
- [docs/ROADMAP.md](docs/ROADMAP.md)
- [docs/STANDARDS.md](docs/STANDARDS.md)
- [docs/GLOSSARY.md](docs/GLOSSARY.md)

## Acceptable Contributions

- New or improved handbook chapters.
- Architecture clarifications.
- ADR proposals and decision index updates.
- Glossary additions that improve terminology consistency.
- Diagram source files and diagram conventions.
- Documentation templates, prompts, and review checklists.
- Corrections to inaccurate or ambiguous material.

## Documentation Requirements

Every contribution must:

- Use clear engineering language.
- Include enough context for future readers.
- Preserve established terminology.
- Update related indexes, references, and changelog entries when scope changes.
- Avoid placeholder text, lorem ipsum, incomplete sections, and unsupported implementation claims.

## Pull Request Requirements

Pull requests should explain:

- The problem or gap being addressed.
- The files changed.
- The impact on roadmap, architecture, terminology, or governance.
- Any follow-up work that should be tracked separately.

## Review Standard

Reviewers should evaluate accuracy, structure, completeness, terminology, security implications, architectural consistency, and long-term maintainability. Style-only feedback should be secondary to correctness and clarity.
