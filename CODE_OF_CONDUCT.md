# Code of Conduct

## Purpose

The AIFlow OS Engineering Handbook exists to support serious technical collaboration. Contributors are expected to create an environment where architecture, documentation, and review work can be performed with clarity, respect, and accountability.

## Expected Behavior

- Use professional, precise, and respectful language.
- Critique ideas, documents, and decisions rather than individuals.
- Make architectural disagreements explicit and evidence-based.
- Welcome contributions from engineers with different backgrounds, domains, and levels of experience.
- Give reviewers enough context to evaluate technical changes.
- Preserve the integrity of the handbook by avoiding unsupported claims, filler content, and vague assertions.

## Unacceptable Behavior

- Harassment, insults, intimidation, or discriminatory language.
- Personal attacks in reviews, issues, discussions, or commits.
- Publishing private information without permission.
- Deliberately misrepresenting technical evidence or project status.
- Repeatedly bypassing documented review, security, or governance processes.

## Enforcement

Maintainers may remove content, request revisions, restrict participation, or close discussions that violate this code. Enforcement decisions should be documented when they materially affect repository governance or contributor access.

## Reporting

Report conduct concerns through the security or maintainer contact process defined in [SECURITY.md](SECURITY.md). Reports should include relevant links, dates, and a factual description of the behavior.
