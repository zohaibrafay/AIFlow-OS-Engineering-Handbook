### Added

- Established Sprint 002 handbook governance and documentation framework.
- Expanded the constitution into the governing document for project, architecture, documentation, review, versioning, and change management.
- Expanded master context with current version, sprint state, repository statistics, decisions, risks, priorities, milestones, and next sprint preview.
- Rewrote the roadmap as a ten-volume product and documentation roadmap with dependencies, exit criteria, estimated chapters, and page ranges.
- Expanded architecture documentation for repository architecture, lifecycle, review, publishing, cross-referencing, and future implementation relationship.
- Expanded documentation standards for Markdown, headings, naming, diagrams, metadata, references, tables, code blocks, callouts, glossary usage, and consistency.

- Added documentation lifecycle and review process guides.
- Added Mermaid diagrams for repository structure, documentation lifecycle, handbook lifecycle, volume relationships, review workflow, publishing workflow, and future platform relationship.
- Expanded Volume 1 into a structured set of foundation parts covering constitution, product, engineering, architecture, development, standards, infrastructure, and roadmap.
- Improved GitHub issue and pull request templates for documentation, review, ADR proposals, feature requests, and bug reports.

### Changed

- Rewrote README for publication-quality repository orientation.
- Rewrote AI agent instructions to enforce required reading, scope boundaries, update rules, and sprint completion behavior.
